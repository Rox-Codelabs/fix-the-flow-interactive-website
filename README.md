In deze sprint heb ik me voornamelijk beziggehouden met het leren van **JavaScript** en het implementeren van verschillende interacties op de website.

### 1. Hamburger Menu
- Het bestaande **CSS-only hamburger menu** is omgezet naar een menu dat opent en sluit met JavaScript.
- Dankzij JavaScript is er nu een **transition** toegevoegd, waardoor het menu vloeiend opent en sluit.

### 2. Scrollende Header
- De header verandert van stijl bij scrollen, net zoals op de huidige website.
- Bij volledig omhoog scrollen verandert de header weer terug naar de oorspronkelijke staat.

### 3. Popup Filter
- Er is een **popup gemaakt** voor het zoeken naar specifieke artikelen.
- In deze popup kunnen gebruikers **3 vragen beantwoorden**, waarna de artikelen gefilterd zouden moeten worden (filterfunctie nog niet volledig ge√Ømplementeerd).
- Wat wel werkt:
  - **Filter count**: toont hoeveel filters zijn toegepast.
  - **Article count**: toont hoeveel artikelen zichtbaar zijn in het overzicht.
- Toekomstige plannen:
  - Filters uit de popup ook zichtbaar maken in de **linker kolom**.

### 4. Automatische Tekstafbreking in Artikel Cards
- De teksten in de artikel cards worden automatisch afgebroken na een bepaald aantal woorden.
- Afgebroken teksten eindigen met `...`, dit gebeurt **dynamisch via JavaScript**, niet handmatig.

### 5. User Story en Wireflow
- De **filter interactie** is uitgewerkt via:
  1. Het schrijven van een **user story**.
  2. Het maken van een **wireflow**.
  3. Het implementeren met een **JavaScript 3-stappenplan**.
- Aan de hand van de user story en interactie is ook een **scenario geschreven**.
- Dit scenario is gebruikt om een **user test** uit te voeren.
Dit kun je terug lezen in [**Issue 8**](https://github.com/Rox-Codelabs/fix-the-flow-interactive-website/issues/8)

### 6. Navigatie en Labels
- Tijdens de lessen over **navigeren en labels** heb ik geleerd dat **icoontjes labels nodig hebben** voor duidelijkheid.
- Labels zijn al toegepast op sommige onderdelen, bijvoorbeeld in de popup.
- In de header zijn de labels nog niet toegepast, omdat deze **identiek moet blijven aan de huidige website.
